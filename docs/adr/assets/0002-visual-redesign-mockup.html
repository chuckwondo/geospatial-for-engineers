$ Who this is for

You already write software. You are comfortable with JSON, the command line, data structures, and reading an unfamiliar API. What you don't have is a geospatial or earth-observation (EO) background: terms like CRS, projection, raster, coverage, footprint, COG, and Zarr go by without quite landing, and most tutorials assume the domain knowledge you're missing while carefully explaining the code you already understand.

This series inverts that assumption. It spends its words on the domain (what these concepts mean and why they exist) and trusts you with the code. It is the on-ramp I wanted when I started doing EO work without the background.

CoverageJSON Uncovered

A complete, ground-up guide to reading coverage data on the web: every concept, from your first document to the edge cases.

This guide is for people new to CoverageJSON (often shortened to CovJSON) who need to understand what it is, why it exists, and how to read a CovJSON document, starting from no prior knowledge of geospatial data formats. It assumes you can read JSON and nothing else. The geospatial ideas it relies on are introduced as they come up and explained just far enough to read CoverageJSON with confidence.

§ What a "coverage" actually is

The single most important idea in this entire guide is this:

! important

A coverage maps positions in space and time to values: one function per property, all sharing the same domain.

inote / tip △warning ╳caution

A coverage is exactly that kind of function, made concrete: its domain is a collection of positions in space and time, and its range is the set of values measured or modeled at those positions. Ask a coverage "what is the value here?" and it answers. Hold onto the words domain and range: they are the literal names of the two main parts of every CoverageJSON document.

A first document

◉ point-coverage.json json 
{
  "type": "Coverage",
  "domain": {
    "type": "Domain",
    "domainType": "Point",
    "axes": {
      "x": { "values": [12.5] },
      "y": { "values": [45.3] },
      "t": { "values": ["2024-07-01T12:00:00Z"] }
    }
  },
  "parameters": {
    "temperature": {
      "type": "Parameter",
      "unit": { "label": { "en": "degree Celsius" }, "symbol": { "value": "Cel" } }
    }
  },
  "ranges": {
    "temperature": {
      "type": "NdArray",
      "dataType": "float",
      "values": [18.4]
    }
  }
}
